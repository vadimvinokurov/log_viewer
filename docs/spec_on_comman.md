## 1. Общее описание продукта

### 1.1. Назначение
Приложение для просмотра, фильтрации и анализа лог-файлов с Vim-подобным интерфейсом: модальный подход, командная строка с префиксом `:`, навигация в стиле Vim.

## 2. Функциональные требования

### 2.1. Типы поиска/фильтрации

**Важно:** Для plain и simple режимов вводимый текст НЕ требует экранирования. Для regex режима используется стандартный синтаксис регулярных выражений.

**По умолчанию:** все команды поиска, фильтров и хайлайтов работают в режиме case insensitive (игнорируют регистр). Для учёта регистра используйте флаг `/cs`.

| Тип | Команда | Пример | Описание |
|-----|---------|--------|----------|
| **Plain** | `:s`, `:f`, `:h` | `:f Some text` | Поиск подстроки |
| **Regex** | `:sr`, `:fr`, `:hr` | `:fr error_\d+` | Регулярное выражение |
| **Simple** | `:ss`, `:fs`, `:hs` | `:fs "ERROR" AND "timeout"` | Булева логика |

**Флаги (добавляются через `/`):**

| Флаг | Описание |
|------|----------|
| `/cs` | Case sensitive — учитывать регистр |
| `/color=<color>` | Цвет для хайлайта |

**Формат команды:**

```
:команда текст                           # без флагов (пробел — разделитель)
:команда/флаги/текст                     # с флагами (слеш после флагов)
```

**Примеры:**

```
:f Some text                    # без флагов, ищем "Some text"
:f/cs/Some text                 # с флагом cs, ищем "Some text"
:fs "ERROR" AND "timeout"       # case insensitive, simple
:h/color=red/Some text          # case insensitive, цвет red
:h/cs/color=yellow/Some text    # case sensitive, цвет yellow
:hr/error_\d+                   # case insensitive, regex
```
**Антипример:**
```
:f//path/to/file               # пустых флагов быть не может - ошибка
```

**Примечание:** Пустых флагов НЕ ДОЛЖНО быть: f//path/to/file - ошибка

**Примечание:** Первый пробел после команды — разделитель. Если текст начинается с пробела, используйте второй пробел: `:f  text` (ищет " text" с ведущим пробелом).

#### Simple mode синтаксис:

```
Операторы: AND, OR, NOT (case-insensitive)
Группировка: ( )
Текст: "двойные кавычки"
Приоритет: NOT > AND > OR (как в булевой алгебре)
```

**Примеры Simple mode:**
```
:ss "ERROR"
:ss "ERROR" OR "WARNING"
:ss "ERROR" AND "timeout"
:ss "ERROR" AND ("timeout" OR "connection") NOT "debug"
```

### 2.2. Режимы работы

**Модальный подход (как в Vim):**

| Мода | Активация | Назначение |
|------|-----------|------------|
| **Normal mode** | По умолчанию | Навигация по логам |
| **Command mode** | `:` | Ввод команд |
| **Search mode** | `/` или `?` | Поиск вперёд/назад |

**Три режима работы с паттернами:**

| Режим | Команда | Поведение |
|-------|---------|-----------|
| **Search** | `:s`, `:sr`, `:ss`| Навигация к совпадениям |
| **Filter** | `:f`, `:fr`, `:fs` | Отображение только совпадающих строк |
| **Highlight** | `:h`, `:hr`, `:hs` | Подсветка совпадений цветом |

**Поиск через `/` и `?`:** работает как же как `:s`, это считай alias для `:s`. 

**Навигация по совпадениям:**

| Клавиша | Действие |
|---------|----------|
| `n` | Следующее совпадение |
| `N` | Предыдущее совпадение |

**Комбинирование:**
- Несколько **фильтров** комбинируются через **OR** (строка показывается если попадает хоть в один фильтр)
- Несколько **хайлайтов** отображаются одновременно разными цветами
- **Search** работает в рамках текущего фильтра и категорий

### 2.3. Навигация и команды

#### 2.3.1. Навигация (Normal mode)

| Клавиша | Действие |
|---------|----------|
| `Esc` | Отмена / возврат в Normal mode |

**Поиск:**

| Клавиша | Действие |
|---------|----------|
| `/text` | Поиск вперёд (работае так же как и `:s`) |
| `?text` | Поиск назад (работае так же как и `:s`) |
| `n` | Следующее совпадение |
| `N` | Предыдущее совпадение |

#### 2.3.2. Команды (Command mode `:`)

**Поиск:**

| Команда | Действие |
|---------|----------|
| `:s <text>` | Поиск plain (case insensitive) |
| `:s/cs/<text>` | Поиск plain (case sensitive) |
| `:sr <pattern>` | Поиск regex (case insensitive) |
| `:sr/cs/<pattern>` | Поиск regex (case sensitive) |
| `:ss <text>` | Поиск simple (case insensitive) |
| `:ss/cs/<text>` | Поиск simple (case sensitive) |

**Фильтры:**

| Команда | Действие |
|---------|----------|
| `:f <text>` | Фильтр plain (case insensitive) |
| `:f/cs/<text>` | Фильтр plain (case sensitive) |
| `:fr <pattern>` | Фильтр regex (case insensitive) |
| `:fr/cs/<pattern>` | Фильтр regex (case sensitive) |
| `:fs <text>` | Фильтр simple (case insensitive) |
| `:fs/cs/<text>` | Фильтр simple (case sensitive) |
| `:rmf <text>` | Удалить фильтр (по точному совпадению) |
| `:rmf` | Очистить все фильтры |

**Highlights:**

| Команда | Действие |
|---------|----------|
| `:h <text>` | Хайлайт plain (case insensitive, дефолтный цвет) |
| `:h/cs/<text>` | Хайлайт plain (case sensitive, дефолтный цвет) |
| `:h/color=red/<text>` | Хайлайт plain (case insensitive, цвет red) |
| `:h/cs/color=red/<text>` | Хайлайт plain (case sensitive, цвет red) |
| `:hr <pattern>` | Хайлайт regex (case insensitive, дефолтный цвет) |
| `:hr/color=yellow/<pattern>` | Хайлайт regex (case insensitive, цвет yellow) |
| `:hs <text>` | Хайлайт simple (case insensitive, дефолтный цвет) |
| `:rmh <text>` | Удалить хайлайт (по точному совпадению) |
| `:rmh` | Очистить все хайлайты |


**Другое:**

| Команда | Действие |
|---------|----------|
| `:n` | Следующее совпадение поиска |
| `:N` | Предыдущее совпадение поиска |

#### 2.3.3. Флаги

**Доступные флаги:**

| Флаг | Описание |
|------|----------|
| `/cs` | Case sensitive — учитывать регистр |
| `/color=<color>` | Цвет для хайлайта |

**Формат для команд :s,:sr,:sf,:f,:fr,:fs,:h,:hr,:hs:**
```
:команда <текст>                    # без флагов
:команда/флаги/<текст>               # с флагами
```

**Примеры:**
```
:f Some text                    # case insensitive, plain
:f/cs/Some text                 # case sensitive, plain
:h/color=red/Some text          # case insensitive, цвет red
:h/cs/color=yellow/Some text   # case sensitive, цвет yellow
```

**Дефолтный цвет для хайлайтов:** первый цвет из списка (red).

**Именованные цвета (базовые):**
- `red`, `green`, `blue`, `yellow`, `cyan`, `magenta`, `white`, `black`, `orange`, `purple`, `pink`, `brown`, `grey`

**Кастомные цвета:**
- HEX: `/color=#FF5733`

#### 2.3.4. Удаление фильтров/хайлайтов

Удаление по полному соответствию команды (включая флаги).

**Примеры:**
```
# Добавление                → Удаление
:f Some text                → :rmf Some text                ✅
:f/cs/Some text             → :rmf/cs/Some text             ✅
:h Some text                → :rmh Some text                ✅
:h/color=red/Some text      → :rmh/color=red/Some text     ✅
```

#### 2.3.6. История команд

**Навигация (`↑`/`↓` в Command mode):**
- Прокрутка истории команд
- Не последовательные дубликаты сохраняются

#### 2.3.7. Просмотр активных фильтров и хайлайтов

```
:lsf                     # Список фильтров
:lsh                     # Список хайлайтов
```

**Формат вывода `:lsf`:**
```
Active filters (2):
  1. :f Some text
  2. :f/cs/Some text 
```

**Формат вывода `:lsh`:**
```
Active highlights (2):
  1. :h Some text
  2. :h/cs/Some text 
```

### 2.5. Система пресетов

#### 2.5.1. Хранение
- **Локация:** `~/.logviewer/presets/<имя>.yaml`
- **Формат:** YAML
- **Тип:** Глобальные (не привязаны к файлу)

#### 2.5.2. Структура пресета
```yaml
name: my-errors
filters:
  - :fs "ERROR" AND "timeout"
  - :fr/error_\d+
highlights:
  - :h/color=red/ERROR
  - :h/color=yellow/WARNING
disabledCategories:
  - network/dns
  - application/debug
```

#### 2.5.3. Применение пресета

**Partial apply:** пресет изменяет только указанные категории и фильтры/хайлайты.

| Правило | Поведение |
|---------|-----------|
| **Категории** | Применяются только указанные в `disabledCategories` |
| **Неуказанные категории** | Состояние не меняется |
| **Несуществующие категории** | Игнорируются (без ошибки) |
| **Несуществующий пресет** | Ошибка: `❌ Preset 'name' not found` |
| **Фильтры/хайлайты** | Добавляются к текущим, дубликаты игнорируются |

**Пример:**
```
Текущее состояние: network/dns = enabled, network/transport = enabled

Пресет (my-errors.yaml):
disabledCategories:
  - network/dns

После применения: network/dns = disabled, network/transport = enabled (не изменился)
```

## 4. Нефункциональные требования

### 4.1. Производительность
| Требование | Значение |
|------------|----------|
| Загрузка файла 1GB | < 10 сек |
| Применение фильтра | < 100 мс |
| Переключение темы | < 50 мс |
| Подсчёт статистики | Асинхронно, не блокирует UI |

### 4.2. Совместимость
| Платформа | Поддержка |
|-----------|-----------|
| **Linux** | ✅ Полная |
| **macOS** | ✅ Полная |
| **Windows** | ✅ Полная (PowerShell, CMD) |
| **Python** | 3.9+ |

### 4.4. История команд
- **Хранение:** `~/.logviewer/history.json`
- **Размер:** Определяется `historySize` в settings.json (по умолчанию 100)
- **Доступ:** Только `↑` / `↓` в Command mode
- **Навигация по логам:** Отдельные клавиши (`↑`/`↓` в Navigation mode, `Page Up`/`Page Down`, `g`/`G`)
