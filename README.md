# Log Viewer v2.0

TUI-приложение для просмотра, фильтрации и поиска по лог-файлам с Vim-стилем навигации.

Построено на [Textual](https://textual.textualize.io/) + [Rich](https://rich.readthedocs.io/).

## Установка

```bash
# Клонировать репозиторий
git clone <repo-url>
cd log_viewer

# Установить зависимости (требуется uv)
uv sync

# Запустить
uv run log-viewer path/to/logfile.log
```

### Запуск без установки

```bash
uv run python -m log_viewer.tui.app path/to/logfile.log
```

### Сборка исполняемого файла

```bash
uv run pyinstaller packaging/log-viewer.spec
```

## Формат логов

Парсер ожидает формат:

```
timestamp category [LOG_LEVEL] message
```

| Поле | Описание |
|------|----------|
| `timestamp` | Дата/время (часть после `T` отображается как время) |
| `category` | Иерархическая категория (разделитель `/`) |
| `LOG_LEVEL` | `LOG_CRITICAL`, `LOG_ERROR`, `LOG_WARNING`, `LOG_INFO`, `LOG_DEBUG`, `LOG_TRACE` |
| `message` | Текст сообщения |

Если `LOG_*` не найден — строка парсится с уровнем `INFO`.


## Навигация (Vim-стиль)

В режиме Normal (по умолчанию):

| Клавиша | Действие |
|---------|----------|
| `j` / Стрелка вниз | Строка вниз |
| `k` / Стрелка вверх | Строка вверх |
| `Ctrl+D` | Полстраницы вниз |
| `Ctrl+U` | Полстраницы вверх |
| `gg` | В начало файла |
| `G` | В конец файла |
| `N` + `G` | Перейти к строке N (например, `42G`) |
| `yy` | Скопировать текущую строку в буфер обмена |
| `:` | Войти в режим команды |
| `/` | Поиск вперёд |
| `?` | Поиск назад |
| `n` | Следующее совпадение поиска |
| `N` | Предыдущее совпадение поиска |
| `q` | Выход |

## Режимы поиска

Все команды поиска, фильтрации и подсветки поддерживают три режима сопоставления:

| Суффикс | Режим | Описание |
|---------|-------|----------|
| (без суффикса) | Plain | Поиск подстроки |
| `r` | Regex | Регулярное выражение |
| `s` | Simple | Булева логика (`AND`, `OR`, `NOT`) |

### Флаги

Команды поиска/фильтрации/подсветки поддерживают флаги в формате `/flags/pattern`:

| Флаг | Описание |
|------|----------|
| `cs` | Учитывать регистр (case-sensitive) |
| `color=<цвет>` | Цвет подсветки: `red`, `green`, `yellow`, `blue`, `magenta`, `cyan` |

## Команды

Все команды вводятся через `:` в режиме Normal.

### Поиск

| Команда | Описание | Пример |
|---------|----------|--------|
| `:s <text>` | Поиск подстроки | `:s error` |
| `:sr <regex>` | Поиск по регулярному выражению | `:sr error_\d+` |
| `:ss <query>` | Поиск с булевой логикой | `:ss "ERROR" AND "timeout"` |
| `:s/cs/error` | Поиск с учётом регистра | |
| `/text` | Быстрый поиск вперёд | `/timeout` |
| `?text` | Быстрый поиск назад | `?timeout` |
| `n` | Следующее совпадение | |
| `N` | Предыдущее совпадение | |

### Фильтрация

Фильтры скрывают строки, не соответствующие паттерну. Можно комбинировать несколько фильтров.

| Команда | Описание | Пример |
|---------|----------|--------|
| `:f <text>` | Фильтр по подстроке | `:f error` |
| `:fr <regex>` | Фильтр по regex | `:fr error_\d+` |
| `:fs <query>` | Фильтр с булевой логикой | `:fs "ERROR" OR "WARNING"` |
| `:f/cs/Error` | Фильтр с учётом регистра | |
| `:rmf` | Удалить все фильтры | |
| `:rmf <pattern>` | Удалить фильтр по паттерну | `:rmf error` |
| `:lsf` | Показать список фильтров | |

### Подсветка

Подсветка выделяет совпадающий текст цветом (по умолчанию — красный).

| Команда | Описание | Пример |
|---------|----------|--------|
| `:h <text>` | Подсветить подстроку | `:h timeout` |
| `:hr <regex>` | Подсветить regex | `:hr \d{3,}` |
| `:hs <query>` | Подсветка с булевой логикой | `:hs "ERROR" AND "file"` |
| `:h/color=green/success` | Указать цвет | |
| `:h/cs/Error` | С учётом регистра | |
| `:rmh` | Удалить все подсветки | |
| `:rmh <pattern>` | Удалить подсветку по паттерну | `:rmh timeout` |
| `:lsh` | Показать список подсветок | |

### Категории

Категории образуют иерархическое дерево (разделитель `/`). Можно включать/выключать целые ветки.

| Команда | Описание | Пример |
|---------|----------|--------|
| `:lscat` | Показать все категории | |
| `:cate <path>` | Включить категорию | `:cate HordeMode/game_storage` |
| `:catd <path>` | Выключить категорию | `:catd LEECH` |
| `:catea` | Включить все категории | |
| `:catda` | Выключить все категории | |

### Пресеты

Пресеты сохраняют текущее состояние фильтров, подсветок и выключенных категорий.

| Команда | Описание | Пример |
|---------|----------|--------|
| `:preset save <name>` | Сохранить текущее состояние | `:preset save debug-session` |
| `:preset load <name>` | Загрузить пресет | `:preset load debug-session` |
| `:rmpreset <name>` | Удалить пресет | `:rmpreset debug-session` |
| `:lspreset` | Список сохранённых пресетов | |

### Файлы и тема

| Команда | Описание | Пример |
|---------|----------|--------|
| `:open <path>` | Открыть файл или URL | `:open /var/log/syslog` |
| `:open <url>` | Загрузить лог по HTTP(S) | `:open https://example.com/log.txt` |
| `:reload` | Перезагрузить текущий файл | |
| `:theme dark` | Тёмная тема | |
| `:theme light` | Светлая тема | |
| `:q` | Выход | |

## Булева логика (Simple mode)

Команды с суффиксом `s` (`:ss`, `:fs`, `:hs`) поддерживают булевы выражения:

- Термы в двойных кавычках: `"error"`
- Операторы: `AND`, `OR`, `NOT`
- Группировка скобками: `()`
- Приоритет: `NOT` > `AND` > `OR`

Примеры:

```
:ss "ERROR" AND "timeout"
:fs "ERROR" OR "WARNING"
:fs NOT "DEBUG"
:hs ("CRITICAL" OR "ERROR") AND NOT "network"
```

## Конфигурация

Настройки хранятся в `~/.logviewer/settings.json`. Пресеты — в `~/.logviewer/presets/`.
